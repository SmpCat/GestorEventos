# Reglas Finales de Sesión
Siempre que se finalice o se ponga en pausa una sesión de trabajo con el usuario, debo asegurarme de hacer estas dos tareas obligatorias:
1. **Limpieza de Entorno**: Limpiar código muerto, revisar si hay errores silenciosos de compilación (como tipos TypeScript implícitos) y dejar el proyecto impecable sin etiquetas ni funciones redundantes.
2. **GitHub**: Hacer un `git commit` descriptivo de todo lo realizado en esa sesión de trabajo y hacer un `git push` a GitHub para dejar el código siempre sincronizado y a salvo.
